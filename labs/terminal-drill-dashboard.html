NAVIGATION
Objective
Every time you SSH into a remote server or open a Docker container shell, these are the first commands you run. Goal: type all 8 without thinking. Target: 5 reps each minimum. Critical ones need 7+.
FILE OPS
Objective
You will manage files on remote servers daily — creating configs, copying backups, cleaning up old models. Goal: execute every file operation without checking syntax. Never confuse mv and cp.
REDIRECTING
Objective
Confusing > with >> can destroy a config file. One overwrites, one appends. Drill this until the difference is automatic. This is a production safety rule.
READING FILES
Objective
tail -f is the most used MLOps command. You will watch training logs, Docker container logs, and pipeline output live every single day. head lets you inspect datasets without loading them. Master both.
TEXT PROCESSING
Objective
These commands let you inspect and transform data directly from the terminal — no Python needed for quick checks. Key rule: always sort before uniq. Never use uniq alone.
PIPES & SEARCH
Objective
The pipe is the most powerful concept in Linux. Chaining commands is how you diagnose production issues fast. grep + wc + sort + uniq together can answer almost any log question in under 5 seconds.
VIM
Objective
On remote servers, vim is often the only editor available. You must be able to open, edit, save, and exit without hesitation. Getting stuck in vim in front of a team is embarrassing. Drill :wq and :q! until automatic.
